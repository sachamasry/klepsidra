      Klepsidra

      Klepsidra keeps the contexts that define your domain and business logic.

      Klepsidra.Accounts

      The Accounts context.

      Klepsidra.Accounts.User

      Define a schema for the User entity, recording authorised system users, for authorisation, ownership, auditing and logging purposes.

      Klepsidra.BusinessPartners

      The BusinessPartners context.

      Klepsidra.BusinessPartners.BusinessPartner

      Defines a schema for the Business Partners entity, recording customers and suppliers of the busines.

      Klepsidra.BusinessPartners.Note

      Defines the data schema for the business partnersNote entity, annotations of business partners.

      Klepsidra.Categorisation

      The Categorisation context provides a way to categorise entities within the application.

      Klepsidra.Categorisation.JournalEntryTags

      Defines a schema for the JournalEntryTags entity, used to create a many-to-many relationship between journal entries and tags.

      Klepsidra.Categorisation.ProjectTags

      Defines a schema for the ProjectTags entity, used to create a many-to-many relationship between projects and tags.

      Klepsidra.Categorisation.Tag

      Defines a schema for the Tags entity, used for categorising timed activities with free form tags.

      Klepsidra.Categorisation.TimerTags

      Defines a schema for the TimerTags entity, used to create a many-to-many relationship between timers and tags.

      Klepsidra.Cldr

      Provides the core functions to retrieve and manage the CLDR data that supports formatting and localisation.

      Klepsidra.Cldr.AcceptLanguage

      Parses HTTP Accept-Language header values as defined in rfc2616.

      Klepsidra.Cldr.Calendar

      Data functions to retrieve localised calendar information.

      Klepsidra.Cldr.Currency
      Klepsidra.Cldr.Date
      Klepsidra.Cldr.Date.Interval

      Interval formats allow for software to format intervals like "Jan 10-12, 2008" as a shorter and more natural format than "Jan 10, 2008 - Jan 12, 2008". They are designed to take a start and end date, time or datetime plus a formatting pattern and use that information to produce a localized format.

      Klepsidra.Journals

      The Journals context.

      Klepsidra.Journals.JournalEntry

      Defines the journal_entries schema needed to record a generic set of journaling needs, from the deeply personal, to commercial.

      Klepsidra.Journals.JournalEntryTypes

      Defines the JournalEntryTypes schema and functions needed to categorise the type of journal entries recorded.

      Klepsidra.KnowledgeManagement

      The KnowledgeManagement context.

      Klepsidra.KnowledgeManagement.Annotation

      Defines the annotations schema, for recording annotations and quotes from source material.

      Klepsidra.Localisation

      The Localisation context.

      Klepsidra.Localisation.Language

      Defines a schema for the Languages entity, listing the languages of the world.

      Klepsidra.Locations

      Location utilities for countries, subdivisions and cities.

      Klepsidra.Locations.AdministrativeDivisions1

      Defines a schema for the AdministrativeDivision1 entity, listing GeoNames' country code and administrative division 1 codes, data that is used in their cities database.

      Klepsidra.Locations.AdministrativeDivisions2

      Defines a schema for the AdministrativeDivision2 entity, listing GeoNames' country code, administrative division 1, and administrative division 2 codes, data that is used in their cities database.

      Klepsidra.Locations.City

      Defines a schema for the City entity, used to select cities of the world.

      Klepsidra.Locations.Continent

      Defines a schema for the Continents entity, listing continents of the world.

      Klepsidra.Locations.Country

      Defines a schema for the Country entity, listing the countries of the world.

      Klepsidra.Locations.FeatureClass

      Defines a schema for the FeatureClass entity, listing GeoNames' feature classes, used for categorising locations around the world. This data is used in their cities database.

      Klepsidra.Locations.FeatureCode

      Defines a schema for the FeatureCode entity, listing GeoNames' feature classes and codes, categorising locations around the world. This data is used in their cities database.

      Utilities module defining mathematical functionality, commonly used all across the project.
