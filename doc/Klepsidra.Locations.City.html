      Klepsidra.Locations.City (Klepsidra v0.1.3)

      Defines a schema for the City entity, used to select cities of the world.

      This is not meant to be a user-editable entity, imported on a periodic basis from the Geonames project, specifically the cities500.zip file, all cities with a population greater than 500.

      @spec city_option_for_select(city_id :: Ecto.UUID.t()) :: %{
  label: String.t(),
  value: Ecto.UUID.t() | String.t()
}

      Constructs an HTML select option for a single city entity, for use by the live_select live component.

      Given a current location_id, a foreign key reference to a city in the locations_cities table, calls the get_city_territory_and_country/1 query, obtaining necessary fields to construct a full, unambiguous, city name.

      Returns

      Returns a single map:

      %{
  label: << city_name, territory - country name >>,
  value: << city_id (UUID) >>

      Examples 

      iex> city_option_as_html_select(UUID)
%{label: "...", value: "UUID"}

iex> city_option_as_html_select(123)
%{label: "", value: ""}
